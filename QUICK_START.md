# Quick Start Guide - 5 Minutes to Your Portfolio! ⚡

## Step 1: Open the Data File (1 min)
Open `js/data.js` in any text editor

## Step 2: Update Your Info (2 min)

### Personal Details
```javascript
name: "Uday Raut",  // ✏️ Already set!
email: "uday.raut@example.com",  // ✏️ Change this
phone: "+91 XXXXX XXXXX",  // ✏️ Change this
```

### Social Links
```javascript
github: "https://github.com/YOUR_USERNAME",  // ✏️ Add your username
linkedin: "https://linkedin.com/in/YOUR_USERNAME",  // ✏️ Add your username
```

## Step 3: Add Your Projects (2 min)

Find the `projects` array and update:

```javascript
{
    title: "My Awesome Project",  // ✏️ Your project name
    description: "What it does",  // ✏️ Brief description
    technologies: ["HTML", "CSS"],  // ✏️ Tech you used
    github: "your-github-link",  // ✏️ Your repo
    live: "your-live-link"  // ✏️ Live demo (optional)
}
```

## Step 4: Open in Browser
Double-click `index.html` - Done! 🎉

## What to Edit Later

- **About section**: Tell your story
- **Skills**: Add/remove skills and adjust levels
- **Experience**: Add internships, education, etc.
- **Projects**: Keep adding as you build more!

## Pro Tips 💡

1. **Save often** - After each change, save and refresh browser
2. **Check syntax** - Make sure commas and quotes are correct
3. **Start simple** - Add 2-3 projects first, expand later
4. **Test mobile** - Open browser dev tools (F12) and test responsive view

## Common Mistakes to Avoid ❌

- Missing commas between array items
- Forgetting quotes around text
- Breaking the JSON structure
- Using special characters without escaping

## Need Help?

- **Syntax error?** Check for missing commas or quotes
- **Not loading?** Open browser console (F12) to see errors
- **Styling issues?** Make sure CSS file is in the right folder
